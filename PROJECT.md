# Project: Library Management Application

## Objective
Create a responsive library management application that allows librarians to keep track of the books in their library.

## Requirements
1. The application should be built using the Angular framework, HTML, JavaScript, and CSS or SCSS.
2. The application should include the following features:
- A list of all available books.
- Detailed book view with additional information, which can be implemented as a modal, a separate route, or any other similar approach.
- The ability to add, edit, and delete books. The add/edit form should include sensible validation checks.
- A search/filter function that allows users to search for books by name.
3. The initial list of books should be provided from an external JSON file or from within the application code.
4. The application should be responsive and work well on different screen sizes.
5. The solution will be checked based on the following criteria (in order of importance):
- Completed functionality.
- Clean code and maintainability (formatting, conventions, readability, etc.)
- Code design and best practices.
- Additional features.
- Design.
6. It is understandable that this is an exam and does not have to be production-ready. If the candidate leaves things at a certain point where they know it can be improved further, they should write a comment indicating so and how.

## Optional Features (Bonus Points)
The following features are not required but will earn bonus points if implemented:
1. Use a free book API, such as Google Books API or Open Library API, to populate the initial list of books.
2. Deploy the application to a static hosting service, such as GitHub Pages, Netlify, Surge, or Firebase Hosting, and provide a link to the deployed app.
3. Allow users to sort books by author, publication date, or catalog number.
4. Add pagination to the book list.
5. Include a book lending feature that allows librarians to mark books as checked out and checked in.
6. Implement unit tests for the application.

##  Submission
When you have completed the assignment, please do the following:
1. Upload your code to a GitHub repository and make it public.
2. Submit the GitHub repository link in the submission form provided to you.
3. If you implemented any optional features, such as using a book API or deploying the application, please provide relevant links in the submission form.
Please make sure to include a README file in your repository with instructions on how to run your application locally. Additionally, if you encountered any issues or had to make any trade-offs during development, please document them in the README file.
Good luck!
